═══════════════════════════════════════════════════════════════════════════════
                            RELATIONSHIP TYPES / ТИПЫ СВЯЗЕЙ
═══════════════════════════════════════════════════════════════════════════════

  USERS ──(1:N)──> BOOKINGS      Один пользователь может иметь много бронирований
  USERS ──(1:N)──> REVIEWS       Один пользователь может писать много отзывов
  USERS ──(1:N)──> PAYMENTS      Один пользователь может делать много платежей
  USERS ──(1:N)──> HOTELS        Один пользователь может владеть многими отелями

  HOTELS ──(1:N)──> ROOMS        Один отель имеет много номеров
  HOTELS ──(1:N)──> BOOKINGS     Один отель принимает много бронирований
  HOTELS ──(1:N)──> REVIEWS      Один отель может иметь много отзывов
  HOTELS ──(1:N)──> HOTEL_AMENITIES  Один отель имеет много удобств

  ROOMS ──(N:1)──> ROOM_TYPES    Много номеров относятся к одному типу
  ROOMS ──(1:N)──> BOOKING_ROOMS  Один номер может быть в многих бронированиях

  BOOKINGS ──(1:N)──> BOOKING_ROOMS  Одно бронирование может включать много номеров
  BOOKINGS ──(1:N)──> PAYMENTS       Одно бронирование может иметь много платежей
  BOOKINGS ──(1:1)──> REVIEWS        Одно бронирование = один отзыв

  COUNTRIES ──(1:N)──> CITIES      Одна страна имеет много городов
  CITIES ──(1:N)──> HOTELS         Один город имеет много отелей

  AMENITIES ──(1:N)──> HOTEL_AMENITIES  Одно удобство может быть во многих отелях
