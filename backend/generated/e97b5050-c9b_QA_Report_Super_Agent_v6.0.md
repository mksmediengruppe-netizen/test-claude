## 🐛 КРИТИЧЕСКИЕ БАГИ

### 1. Авторизация через API не работает для защищённых endpoints
**Серьёзность:** 🔴 Критический  
**Статус:** FAIL

**Описание:**
После успешной авторизации через `/api/auth/login` получен токен, но при попытке доступа к защищённым API возвращается ошибка 401 Unauthorized.

**Затронутые endpoints:**
- `/api/chats` - 401 Unauthorized
- `/api/agents` - 401 Unauthorized
- `/api/analytics` - 401 Unauthorized
- `/api/settings` - 401 Unauthorized
- `/api/files` - 401 Unauthorized
- `/api/auth/me` - 401 Unauthorized

**Шаги воспроизведения:**
1. POST `/api/auth/login` с `{email: "minimax", password: "qwerty1985"}`
2. Получен токен: `c3ce61c0eb85afffbf043ce5258ac5c496f18238ba877a790558f54c73705f1e`
3. GET `/api/chats` с заголовком `Authorization: Bearer {token}`
4. Ожидается: 200 OK с данными чатов
5. Фактически: 401 Unauthorized

**Ожидаемое поведение:** API должен принимать токен авторизации и возвращать данные

**Фактическое поведение:** Возвращается ошибка 401 даже с валидным токеном
