🔗 Основные связи между таблицами

HOTELS → ROOM_TYPES 1:N (один отель имеет много типов номеров)
ROOM_TYPES → ROOMS 1:N (один тип имеет много номеров)
USERS → BOOKINGS 1:N (один пользователь делает много бронирований)
BOOKINGS → PAYMENTS 1:N (одно бронирование имеет много платежей)
BOOKINGS → REVIEWS 1:1 (одно бронирование = один отзыв)
